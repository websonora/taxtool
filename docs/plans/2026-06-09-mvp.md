# Tax Document Backup Portal MVP Implementation Plan

> **For Hermes:** Use subagent-driven-development skill to implement this plan task-by-task.

**Goal:** Build a local internal web MVP that lets tax-office staff open a prior-year PDF visually, select pages to carry forward, upload current-year PDFs, merge them, and save a final backup PDF.

**Architecture:** FastAPI serves a simple browser UI and JSON/file endpoints. PyMuPDF handles PDF rendering, extraction, and merging. The MVP stores temporary work in local folders and writes outputs under a configurable local `data/output/<year>/` folder; later this can point to the office UNC path.

**Tech Stack:** Python 3.11, FastAPI, PyMuPDF, pytest, plain HTML/CSS/JavaScript.

---

## MVP Scope

### Included now
- Upload/select prior-year PDF.
- Render pages as browser thumbnails.
- Click pages to keep.
- Upload one or more new current-year PDFs.
- Merge selected prior pages plus all pages from new PDFs.
- Save output PDF to selected tax year folder.
- Prevent accidental overwrite by versioning duplicate output names.
- Minimal audit JSONL log.

### Not included yet
- Real login/auth.
- Database client index.
- Network share configuration UI.
- OCR/AI classification.
- Direct scanner integration.
- Public/cloud access.

### Task 2: PDF service core behavior using TDD

**Objective:** Implement PDF page counting, thumbnail rendering, selected-page extraction, and PDF merging.

**Files:**
- Modify: `app/pdf_service.py`
- Test: `tests/test_pdf_service.py`

**Required behavior:**
- `get_page_count(path) -> int`
- `render_page_thumbnail(path, page_number, zoom=0.25) -> bytes`
- `create_pdf_from_selected_pages(source_path, selected_pages, output_path)` uses 1-based page numbers.
- `merge_pdfs(input_paths, output_path)` preserves input order.
- Invalid page selections raise `ValueError`.

**Verification:** `pytest tests/test_pdf_service.py -q` passes.

### Task 3: Safe output storage and audit log using TDD

**Objective:** Implement local output folder creation, duplicate versioning, filename sanitization, and audit append.

**Files:**
- Modify: `app/storage.py`
- Test: `tests/test_storage.py`

**Required behavior:**

- Existing files produce `Name - version 2.pdf`, `Name - version 3.pdf`, etc.
- `append_audit_log(base_dir, record) -> Path` appends JSON line.

**Verification:** `pytest tests/test_storage.py -q` passes.

### Task 4: FastAPI endpoints

**Objective:** Expose endpoints for upload, thumbnails, merge/create, and static UI.

**Files:**
- Modify: `app/main.py`
- Create: `app/templates/index.html`
- Create: `app/static/app.js`
- Create: `app/static/styles.css`
- Create: `tests/test_app.py`

**Required endpoints:**
- `GET /` returns UI.
- `POST /api/prior-pdf` uploads prior PDF and returns `document_id`, filename, page count.
- `GET /api/prior-pdf/{document_id}/thumbnail/{page_number}` returns PNG thumbnail.
- `POST /api/create-backup` accepts document ID, selected pages, tax year, client filename, and current-year PDFs; returns output path and audit data.

**Verification:** `pytest tests/test_app.py -q` passes.
